introduction

roman-encryption is an encryption system loosely based off of the principle of roman numberals. When reading roman numerals, if a smaller value character comes before a larger value one, the smaller value is subtracted from the total value instead of added. You can read more here.

This principle drives roman-encryption: an encrypted number takes form of an array of numbers. In order to evaluate the array into a single number, the numbers are added up in the same fashion as roman numerals: if a number is smaller than the number after it, it is subtracted; otherwise, it is added to the total.

The security in roman-encryption comes from the addition of another factor: a threshold. When evaluating an encrypted array, you first check if a number is smaller than the one after it; if it isn't, then you add the number. However, if the number is smaller than the one after it, you then check if the difference between the number and the following number is greater than a defined "threshold." Only if the difference is larger than the threshold, will you subtract that number from the total value.

This creates a varying and unpredictable change in the values of evaluated arrays. The threshold is used in creating the encrypted array, and without it, it is difficult to decrypt the same array. The real difficulty of breaking encryption without the threshold is unknown and untested; I am not knowledgable enough to perform any such evaluation.

I believe this encryption algorithm is a proof-of-concept for an additional layer of security, perhaps for encrypting a key for a different encryption. One downside is that encrypted data will increase in size, so the practicality of this method is questionable. These same concepts should be applicable across number systems of differing bases as well.



api

encryption

Send a POST request to roman-encryption.herokuapp.com/encrypt. Body data should be in the format of "form-data."

Parameter Definition
number Value to be encrypted.
threshold Threshold to use in encryption; needed in order to decrypt.
size (optional) Length of desired encrypted array. Default value is 50.

decryption

Send a POST request to roman-encryption.herokuapp.com/decrypt. Body data should be in the format of "form-data."

Parameter Definition
array Array of the encrypted number. (Parsed using JSON.parse(). Send as a string in the format of [1, 5, 3, 8]. Spaces are negligible.)
threshold Threshold used in encryption.
